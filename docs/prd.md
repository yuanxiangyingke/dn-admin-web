项目名称： 数字游民产品项目 
产品名称： 数字游民线上服务平台 
日期： 2025年9月5日

---
一、文档概述
1.1 文档目的
本文档旨在详细描述“数字游民线上服务平台”的各项功能、用户体验（UX）和设计规范。本平台包括网页门户、微信小程序、APP（iOS、Android、鸿蒙）以及配套的后台管理系统。本文档是项目团队（产品、设计、开发、测试）的核心参考文件，旨在确保所有相关方对产品目标、功能和需求达成共识，并指导后续的开发工作。
1.2 项目背景与目标
项目背景： 随着远程工作和灵活就业的兴起，数字游民群体迅速壮大。市场存在“线下社区+线上服务”生态的空白，现有解决方案在住宿预订效率、工作机会匹配效率和精准度以及职业发展支持方面存在明显不足。
产品目标： 通过整合“线下社区”与“线上服务”，构建一个集住宿预订、活动发布与参与、在地服务、工作机会匹配与职业发展服务于一体的数字游民全场景解决方案，为用户提供高效、智能、有保障的一站式服务体验。
1.3  目标用户
- 核心用户： 25-35岁知识工作者，如程序员、设计师、自媒体人、一人公司创业者等。
- 潜在用户： 灵活用工企业以及处于职业探索期的潜在数字游民。
二、核心功能模块
1.1 C端模块（网页门户/小程序/APP）
该模块旨在解决用户在寻找和预订住宿、发起与参与活动、寻找工作机会和职业成长时的痛点。
全域搜索框：
- 功能描述：发现、机会模块顶部需要放置搜索框，支持全域搜索
- 交互逻辑：点击搜索框后进入搜索页面，支持导航tab筛选搜索：全部、社区、活动、机会、在地共创、人才，支持组合筛选。
1.1  发现
发现模块包含社区、活动、在地三大模块，点击tab可进入到相应页面
1.1.1 社区预定
1. 功能描述： 
- 高效选择：用户可以通过城市、价格、社区类型、评分等维度选择和浏览全国各地的数字游民社区，降低用户的选择成本。
- 实时房源： 在房间详情页，实时展示可预订的房源状态（如“满房”、“可预订”），避免无效查询。
- 智能预订： 用户可在各平台一键完成在线申请、支付，缩短订房链路。
- 用户评价： 建立多维度、真实的图文评价体系，帮助新用户做决策。
2. 交互逻辑：
  - 首页入口： 在各平台首页提供显眼的入口，引导用户进入“社区”模块。
  - 筛选器： 提供城市、日周月价格（日价格、周价格、月价格）多重筛选条件，支持组合筛选。
  - 列表展示： 以卡片形式展示每个社区，卡片需包含：社区图片、名称、城市、社区标签（需含社区类型，比如创业社区）、价格和用户评分。
  - 房间详情页：以图文形式展示房间详情，下拉页面即可展示房型列表、共居生活、设施、位置、周边、用户评价等tab，点击相应tab即可进入不同页面。
  
1.1.2 活动：
1. 功能描述：
- 活动列表： 用户可按城市、类型（线上/线下）和排序（最新/最热）筛选浏览活动列表。每项活动以卡片形式展示核心信息，如封面、名称、时间、地点、费用（免费、人民币、游民币）、报名用户头像缩略图、报名人数以及报名状态（报名、已满、截止报名）。
- 活动详情与报名： 展示活动的详细信息、报名进度条，并支持用户在线报名。
- 活动发布与管理： 用户可以申请发布自己的活动，并进行线上管理（如报名、签到）。
2. 交互逻辑：
- 活动列表页： 点击活动卡片跳转到活动详情页。
- 活动报名： 免费活动点击“报名”直接成功；付费活动则跳转至支付流程。报名成功后，按钮文案变为“查看入场券”。
- 活动发布： 用户通过底部导航栏“发布”入口进入，填写并提交活动表单。提交后显示“提交成功”页面，进入审核流程。
1.1.3 在地服务模块
1. 功能描述：
  - 社区服务：用户选择“入住社区”后，系统可显示所在社区的在地服务，如智能钥匙、订餐、房间打扫等。
  - 服务入口： 以图文卡片形式展示社区内的各类服务，点击后可进入相应的服务详情页面。
  - 服务详情页：banner图（上个页面相应的封面图），图文详情页介绍，含联系人信息，但不支持付款。
  - 钥匙详情页：定制服务（UI设计参考目前的数字游民小程序-钥匙模块）
2. 交互逻辑：
  - 在地服务页： 用户进入此页面时，需在下拉列表中选择所在的社区。若当前社区没有在地服务，则显示“当前社区无在地服务”提示语。
  - 服务卡片： 点击不同的服务卡片，用户将被引导至相应的服务子页面以完成具体操作。
  
1.2  机会
机会模块包含工作机会、在地共创、服务三大子模块，点击tab可进入到相应页面。
1.2.1 工作机会：
  1.功能描述：
  - 旨在解决用户在寻找工作机会上的痛点。
  - 工作机会列表： 用户可根据职业类型、工作方式、最新等筛选和浏览工作机会列表。列表以信息卡片形式展示，包含发布者用户头像&昵称、职位标题、职位标签、一句话简介、点赞/收藏/浏览数、发布时间等。
  - 工作机会详情： 详细展示工作内容、团队介绍、职责要求、薪酬待遇等信息，并提供“申请职位”入口。还需展示已申请用户的头像缩略图以及展示XX人已申请（参考活动报名详情页），展示项目状态（招募中、进行中、已完成）。
  - 工作机会发布： 用户可填写表单发布工作机会，并支持上传海报和设置项目结算方式（待讨论），增加项目状态筛选框（招募中、进行中、已上线）
  - 数字游民人才库： 用户可创建个人档案，录入技能、兴趣标签、作品集等信息、以及上传简历（在“我的工作”中上传，系统可通过智能算法为用户精准匹配工作机会和创业共创项目。
  - 人才列表：展示用户名片信息——用户头像、昵称、技能、标签等信息列表
  2.交互逻辑：
    - 工作机会列表页： 点击工作机会卡片跳转至工作机会详情页。
    - 申请流程： 点击“申请职位”按钮，支付游民贝进入申请表单填写页，提交后待发布方收到申请（可选择通过或者拒绝），发布方通过后即可在线进行沟通（参考微信加好友）。
    - 发布流程： 通过底部导航栏“发布”入口，选择“发布机会”进入工作机会发布页。
- 1.2.2 在地共创 
  - 功能描述：
    - 设置专门模块，展示与政府、本地企业或者通过社区主理人认证的合作外包项目。
    - 项目列表： 用户可按工作方式、岗位类型等筛选浏览在地共创项目。列表以图文卡片形式展示，带有“官方认证”标识，凸显官方属性。
    - 项目详情与申请： 详细介绍项目背景、团队、职责要求、分成方式等，并提供“申请项目”入口。还需展示项目成员的头像缩略图，不可点击联系。
  - 交互逻辑：
    - 项目列表页： 点击项目卡片跳转至项目详情页。
    - 申请流程：  点击“申请职位”按钮，支付游民贝进入申请表单填写页，提交后待发布方收到申请（可选择通过或者拒绝），发布方通过后即可在线进行沟通（参考微信加好友）
    - 发布流程： 通过底部导航栏“发布”入口，选择“发布共创项目”进入相应发布页面。
- 1.2.3 职业服务：
  - 功能描述：
    - 提供可选的职业相关配套服务，如公司注册、社保、税务、法律咨询等。
    - 服务列表： 顶部Banner轮播介绍服务，下方以图文卡片形式展示各项事业支持服务，包括社保代缴、财税服务、法律服务、创业孵化。
    - 服务入口： 提供清晰的入口，引导用户进入相关的服务页面或联系渠道（参考在地服务）。
  - 交互逻辑：
    - 事业支持页： 用户点击不同的服务卡片，将被引导至相应的服务详情页或联系方式展示页面。
1.3 消息模块
- 功能描述：
  - 该模块作为用户的统一消息中心，聚合了来自平台各模块的通知。
  - 申请联系：用户可点击用户头像进入用户主页（或工作申请）----点击“联系Ta”---支付贝壳---填写申请信息---对方收到申请后可选择通过或者拒绝，通过即可进行沟通。（类似微信的添加好友操作）
  - 消息列表： 每条消息均包含发送者头像、昵称、消息摘要及日期。
- 交互逻辑：
  - 消息列表页： 点击消息列表中的任一消息卡片，即可进入详细的消息对话框，查看完整的沟通内容。
1.4 我的模块
- 功能描述：
  - 该模块是用户的个人中心和数据管理枢纽。
  - 个人名片： 顶部以名片形式展示用户核心信息，包括头像、昵称、职业、工作状态、联系方式（可选）、用户等级、标签信息（MBTI、技能和兴趣），“社区主理人”还需特殊身份标签。
  - 我的主页：个人信息的集合页，展示个人名片、一句话简介、联系方式、技能、兴趣、我的作品集。
  - 我的游民贝：展示我拥有的游民贝数量，游民贝明细（获得和消耗的明细），“购买贝壳”button
  - 多维数据集合： 以Tab形式集中展示用户在平台上的所有活动、工作和入住记录，包括“我的工作”（我发布的、我申请的、我参与的、我收藏的、我的简历）、“我的活动”（我报名的、我发起的、我收藏的）和“我的入住”（待审核、待付款、待入住、待评价、全部订单）。
  - 入住评论： 用户入住后可对住宿体验进行评价，支持多维度评分（如服务、卫生、社区氛围）、图文上传及匿名评论。
  - 智能客服：放在我的主页（开发时间来不及的话用官方客服电话代替）
- 交互逻辑：
  - 我的： 用户点击不同的功能Tab，页面内容会相应切换至该模块下的子页面。
  - 我的主页：点击“我的名片”卡片，进入我的主页。用户分享“我的主页”即可即刻让对方了解自己。
  - 入住评论： 在“我的入住”Tab下的全部订单中，已入住的用户将获得评论入口。提交评论后，系统将显示“评价成功”页面以示感谢。
1.5 账号系统
- 功能描述：
  - 平台提供安全、便捷的登录与注册流程，支持主流社交和通信方式。
  - 登录注册： 支持微信一键登录、手机号一键登录以及短信验证码登录，为新用户提供无缝的注册体验。
- 交互逻辑：
  - 登录页： 用户选择登录方式，完成认证后，系统将自动跳转至主页。对于首次登录的用户，系统将自动完成注册流程，无需额外操作。

1.2 B端模块（后台管理系统）
该模块旨在为甲方提供高效、统一的后台管理工具，以支持C端平台的日常运营。
2.1 空间管理系统
- 功能描述： 面向所有空间（社区、民宿、酒店）的库存与内容管理。
- 核心功能：
  - 社区信息管理（基本信息、图片、定位）。
  - 房型与价格管理，支持动态定价。
  - 实时房源状态更新与同步。
  - 入住申请审批与订单管理。
2.2. 产品管理后台
- 功能描述： 核心运营与数据洞察平台。
- 核心功能：
  - 用户管理： 查看、管理用户资料与行为数据。
  - 人才库管理： 管理数字游民人才库，审核用户提交的档案。
  - 项目与机会管理： 审核并管理工作机会、创业共创项目。
  - 数据统计与洞察： 提供用户活跃度、社区入住率、交易量等关键数据报表，支持大数据分析。
  - 内容管理： 发布、编辑社区活动、公告等内容。
3. 技术架构与账号体系
3.1 账号体系
- 登录方式： 采用手机号、微信一键登录作为主要登录方式，以适应不同平台的用户习惯。
- 账号绑定： 支持在不同平台间绑定同一账号，实现用户数据同步。
3.2 技术架构
- 前端： 网页门户、微信小程序、原生APP（iOS/Android/鸿蒙）。
- 后端： 统一的后台系统，以RESTful API接口为C端各平台提供数据支持。
- 数据库： 支持用户数据、社区数据、订单数据、项目数据等的存储和管理。
4. 用户体系与信用体系
- 功能描述：
  - 平台将建立一套完整的用户等级体系和社区代币体系，以激励用户积极参与社区活动、并为未来精准匹配和增值服务提供数据基础。用户等级与代币分将在用户的个人主页上展示。
用户成长体系：
解决的是 用户身份和等级感，是一种对身份的长线激励
目的是提升用户对平台的情感依赖，从而提升留存率。
用户获得经验值（ XP ）只是为了升级，不能直接使用。
代币体系：
是社区消费，目的是帮助平台形成一个自循环的价值交换市场，从而提升社区的活跃度和质量。

6. 产品发布计划
4.1 第一阶段（上线版本）
- 平台： 微信小程序 + 核心后台管理系统（空间管理、用户管理、订单管理）。
- 核心功能：
  - 社区与空间基础信息展示。
  - 线下空间快捷预订（填写入住申请、人工审批、在线支付）。
  - 社区活动发布与参与。
  - 工作机会与创业共创项目发布、申请与参与（手动筛选）。
4.2 第二阶段（版本迭代）
- 平台： 微信小程序 + 后台管理系统。
- 核心功能：
  - 工作机会智能匹配： 根据技能、经验、兴趣进行智能匹配。
  - 信用体系： 搭建用户信用分体系，并引入到项目匹配流程中。
  - 职业发展服务： 引入公司注册、社保、税务、法律等配套服务。
4.3 第三阶段（平台扩展）
- 平台： 网页门户、原生APP（iOS/Android/鸿蒙）。
- 核心功能：
  - 将第一、二阶段已验证的所有功能同步到网页门户和原生APP上，实现全平台覆盖。
